Sequence 1 — Business software detection during execution

Two sub-scenarios: block before launch + interruption during execution

If the business software is detected before launch, the job is immediately marked as Error and the event is logged. If detected during execution, the current file copy completes first, then the job is stopped via CancellationToken and logged.

Before launch checkBackupService calls BusinessSoftwareDetector.IsRunning() before initializing the job. If true → Error + log + exception to ViewModel.
During execution checkCalled inside OnFileCopied() callback after each file. If detected → finish current CopyFile(), send CancellationToken to strategy, log interruption.
Specification compliance"Finish the current file transfer" — the diagram shows CopyFile() completing normally before the stop signal is sent.
Log entryA dedicated LogEntry type (BusinessSoftwareBlocked / BusinessSoftwareInterrupted) is written via EasyLogAdapter for support traceability.

Sequence 2 — Backup execution with CryptoSoft encryption

Three branches: encrypted file / non-encrypted file / CryptoSoft unavailable

After each file copy, FullBackupStrategy checks whether the file extension is in the configured encrypted extensions list. If yes and CryptoSoft is available, Encrypt() is called and both TransferTimeMs and EncryptionTimeMs are logged. If CryptoSoft fails, EncryptionTimeMs = -1.

Loading...

Encrypted file (.pdf, .docx...)CopyFile() + CryptoSoftAdapter.Encrypt() → LogEntry with TransferTimeMs=128, EncryptionTimeMs=42, IsEncrypted=true.
Non-encrypted file (.log, .tmp...)CopyFile() only → LogEntry with TransferTimeMs=64, EncryptionTimeMs=0, IsEncrypted=false.
CryptoSoft unavailable/errorIsAvailable() = false or Encrypt() throws → EncryptionTimeMs = -1 in log. File is still copied successfully.
EncryptionTimeMs convention0 = no encryption configured, >0 = encryption duration in ms, <0 = CryptoSoft error code.
